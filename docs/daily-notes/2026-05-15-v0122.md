## A — Delta sous chaque mesure

### `<MeasureDelta>` (`src/components/MeasureDelta.tsx`)
Composant partagé. Format :
```
▲ −2 cm (−1.9 %) · depuis 10 sept 2025
```
- Couleur **verte si amélioration**, **rouge si régression** (sens de la performance, géré par `lowerIsBetter`).
- Flèche `▲` / `▼` suit le sens du delta brut (positif/négatif), pas le sens performance.
- Pas de précédent → `Première mesure` en gris pâle.
- `|delta| < epsilon` (défaut 0.05) → `= stable`.
- Prop `theme: 'light' | 'dark'` pour s'adapter au fond (les cards de MesuresTab sont marine — utilisent `theme="dark"`).
- Affiche le pourcentage relatif entre parenthèses si `previous !== 0`.

### Intégration dans MesuresTab
- `MeasureField` enrichi : 4 nouvelles props optionnelles (`previousValue`, `previousDate`, `lowerIsBetter`, `extra`).
- « Mesure précédente » définie contextuellement :
  - Si en mode édition (`editId`) → précédent = bilan juste avant la ligne éditée (`list[idx + 1]`).
  - Si en création → précédent = `list[0]` (le plus récent sauvegardé).
- `lowerIsBetter` activé pour : tour taille / hanche / abdomen / poids / 4 plis cutanés. Les autres (cou, biceps, etc.) restent neutres.

## B — Catégorisation OMS pour tour de taille / hanche / ratio

### Module — `src/lib/norms/who.ts` (nouveau)
Séparé volontairement de `acsm.ts` parce que c'est une **autre échelle conceptuelle** :
- ACSM = catégorisation **fitness** sur 5 catégories (À améliorer → Excellent), basée sur les percentiles de la population.
- OMS = niveau de **risque cardio-métabolique** sur 3 niveaux (Faible / Élevé / Très élevé), basé sur des seuils cliniques.

API :
- `getWaistRisk(value, sex)` → `{ level, thresholds }` ou `null`
- `getRatioRisk(value, sex)` → idem
- `calculateRiskBarPosition(value, thresholds)` → 0-100 % pour la barre 3 segments

Seuils (sources WHO 2008 + Santé Canada) :
- **Tour de taille** : H < 94 / 94-102 / ≥ 102 ; F < 80 / 80-88 / ≥ 88.
- **Ratio T/H** : H < 0.90 / 0.90-1.00 / > 1.00 ; F < 0.80 / 0.80-0.85 / > 0.85.

### `<WaistRiskBar>` (`src/components/WaistRiskBar.tsx`)
- 3 segments flex-1 vert/jaune/rouge (couleurs OMS hardcodées dans `who.ts`).
- Marqueur ▲ positionné selon la valeur.
- Tooltip natif : « OMS — Tour de taille et risque métabolique (WHO 2008) ».
- Label sous la barre : « Élevé · OMS » (couleur cohérente avec le segment).

### Intégration dans MesuresTab
- Champs **tour de taille** et **tour de hanche** : la barre est passée au prop `extra` de `MeasureField`, rendue sous le delta.
- Nouvelle **card Ratio T/H** : apparait dès que taille + hanche sont saisis (calcul `taille / hanche` en cm, indépendant de l'unité d'affichage). Affiche valeur + delta + barre OMS ratio.

## C — Pré-remplissage depuis la dernière mesure

### Bannière + modal — Circonférences
- Bannière gold cliquable au-dessus du formulaire **si** : pas en mode édition ET mesure précédente existe.
- Format : « ⬆ Reprendre les valeurs de la mesure du 10 septembre 2025 — N champs disponibles à pré-remplir » + bouton « Pré-remplir ».
- Clic → `<CircPrefillModal>` :
  - Checkboxes par circonférence + Poids (tous pré-cochés par défaut).
  - Aperçu de la valeur formatée à droite (« 95.0 cm »).
  - Toggle **« Écraser les champs déjà saisis »** (off par défaut — sécurité).
  - Bouton « Pré-remplir (N) » désactivé si rien n'est sélectionné.
- `applyPrefill(selectedCirc, includePoids, overwrite)` : patche `form` + `poids`, ne touche pas aux notes.

### Bannière + modal — Plis cutanés
Idem mais plus simple (4 plis fixes) — `<PlisPrefillModal>`.

## Dette / à garder en tête

- **Couleurs OMS hard-codées** : `#97C459` (vert), `#FAC775` (jaune), `#E24B4A` (rouge) en dur dans `who.ts`. À extraire en CSS vars si on fait un thème dark un jour.
- **MeasureDelta « depuis »** : utilise `formatBilanDate` qui produit « 10 septembre 2025 » — peut prendre 2 lignes sur les cards étroites. Acceptable mais à surveiller à l'usage.
- **Ratio T/H lowerIsBetter** : oui (un ratio bas = bon). MeasureDelta affiche `▼ −0.02 (−2.2 %)` en vert. Cohérent.
- **`applyPrefill` ne reset pas les notes** : si Marie-Eve avait écrit des notes, elles restent. Voulu.
- **Bannière de pré-remplissage cachée en mode édition** : on n'affiche pas le « pré-remplir depuis » quand on édite une ligne existante. Le sens est moins clair en édition. Si Marie-Eve veut l'option, on peut activer ; pour l'instant masqué.
- **Pré-remplissage `lowerIsBetter` non pris en compte pour le toggle Écraser** : le toggle s'applique uniformément à tous les champs sélectionnés. Acceptable.
- **`<CircPrefillModal>` et `<PlisPrefillModal>` partagent une structure très similaire** — pas factorisé. Si on en ajoute un 3e (mesures bilans manuel ?), refactor en `<PrefillModal>` générique.
