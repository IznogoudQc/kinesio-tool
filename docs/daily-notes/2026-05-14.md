# 2026-05-14 — Catégorisation des résultats de tests (ACSM, pluggable)

## Travail réalisé

### Architecture pluggable — `src/lib/norms/`
- **`types.ts`** : `Category` (5 valeurs `A_AMELIORER` → `EXCELLENT`), `CATEGORY_LABELS` (FR), `CATEGORY_COLORS` (Tailwind), `TestKey` (10 tests), `NormsType` (`'acsm' | 'cpafla'`), `NormRange` avec `thresholds: { aAmeliorer, acceptable, bien, tresBien }` et flag `lowerIsBetter` pour inverser la logique (% gras, IMC, tour de taille).
- **`acsm.ts`** : tables **ACSM 11e éd. (2021)** encodées pour VO2max, % gras corporel, push-ups, redressements assis partiels, sit-and-reach (flexion tronc), IMC, tour de taille, endurance du dos (test de Sorensen — hors ACSM mais clinique standard). Saut vertical et puissance des jambes laissés à `null` (pas de table ACSM retenue). Tables par tranches d'âge 20-29 → 70+ × sexe (sauf IMC et tour de taille agnostiques à l'âge).
- **`cpafla.ts`** : placeholder retourne `null` partout. À compléter quand on aura les tables.
- **`index.ts`** : API publique `getCategorization(test, value, age, sex, norms = 'acsm')` qui choisit la bonne table, applique le sens (`lowerIsBetter` ou non), et retourne `null` quand on ne peut pas catégoriser (donnée manquante, hors barème, test non couvert). Helper `computeAge(birthdate)` en bonus.
- **`bilan-keys.ts`** : map des clés du modèle `BilanData` (`vo2max`, `pourcentage_gras`, `imc`, `tour_taille_cm`, `pushups`, `situps`, `saut_vertical_cm`, `puissance_jambes_watts`, `flexion_tronc_cm`, `endurance_dos_sec`) vers `TestKey`.

### Settings
- Clé `categorization_norms` dans la table `settings` (clé/valeur, **pas de migration**). Valeurs : `'acsm'` (défaut) | `'cpafla'`. Validation `zod`.
- IPC : `settings:norms:get` / `settings:norms:set` (`electron/ipc/settings.ts`), exposés via `window.api.settings.{get,set}CategorizationNorms()` (`electron/preload.ts`, `src/env.d.ts`, `src/services/settings.ts`).

### UI — Paramètres (`src/pages/SettingsPage.tsx`)
- Nouvelle section **« Normes de catégorisation »** (icône `Gauge` lucide) entre **Profil** et **SMTP**.
- Deux radios :
  - **ACSM** (American College of Sports Medicine, 11e édition, 2021) — actif.
  - **CPAFLA (à venir)** — quand sélectionné, affiche un encart info : « Les tables CPAFLA seront ajoutées dans une future version. ACSM est utilisé en attendant ».
- Bouton « Enregistrer » classique avec `StatusInline`.

### Intégration Dashboard + BilanDetail
- **`DashboardTab.tsx`** : charge le setting + calcule l'âge du client à partir de `client.birthdate`. Hero `StatCard` accepte une prop `category?: Category | null` qui affiche un badge coloré sous la valeur. Appliqué sur VO2max, IMC, % gras. La carte « Dernier bilan complet » (un seul bilan) passe `categorize` à `BilanForm`.
- **`BilanDetailTab.tsx`** : charge le setting + calcule l'âge, passe `categorize` à `BilanForm`. Le sous-titre indique la norme active (« Catégorisation selon les normes ACSM/CPAFLA — modifiable dans Paramètres ») et remplace l'ancien message « catégorisation CPAFLA en v0.1.7 ».
- **`BilanForm.tsx`** : prop optionnelle `categorize?: (key, value) => Category | null`. Affiche la catégorie sous chaque valeur **uniquement** en `readOnly` (pas pendant l'édition — la valeur change en temps réel).
- **`CategoryBadge.tsx`** : composant partagé qui affiche le label coloré ou un tiret en gris (`text-marine/30` / `text-cream/30` selon le fond) quand `null`.

## Dette / à garder en tête

- **Le test duplique la logique de `getCategorization`** : Node `--test` exige des chemins en `.ts` que `tsc` refuse hors `*.test.ts`. Plutôt que de monter une chaîne d'imports compatible avec les deux, le test ré-implémente la fonction (8 lignes) et importe directement `acsm.ts` / `cpafla.ts` — donc il teste surtout les **tables**, pas la fonction. Acceptable parce que les tables sont le risque réel (faute de frappe sur un seuil) ; la fonction est triviale et exercée par l'UI.
- **CPAFLA** : architecture prête. Quand Marie-Eve confirmera la source (manuel CPAFLA / captures du logiciel actuel / autre), il suffira de remplir `cpafla.ts` — aucune autre modification.
- **Saut vertical / puissance jambes** : pas de table ACSM retenue. Sources possibles si on veut combler : Cooper Institute, Heyward (*Advanced Fitness Assessment and Exercise Prescription*).
- **Endurance du dos** : seuils Sorensen utilisés (60 / 100 / 130 / 170 sec pour les hommes ; 60 / 110 / 140 / 180 pour les femmes) — à valider avec Marie-Eve si elle préfère un autre barème.
