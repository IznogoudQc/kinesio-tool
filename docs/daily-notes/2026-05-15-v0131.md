# 2026-05-15 — Branchement API Anthropic réel (v0.1.31)

La v0.1.30 a livré l'architecture + UI complète avec une réponse mockée. Cette version connecte l'API Anthropic Claude pour de vrai. Décision détaillée dans [[decisions/0008-branchement-api-anthropic]].

## IPC — `electron/ipc/ai.ts` (nouveau)

5 handlers :

- **`ai:has-api-key`** → `boolean`. Lit `keytar.getPassword('kinesio-outils', 'ai-anthropic-api-key')`.
- **`ai:set-api-key`** → écrit dans keytar après validation zod (`min(1).max(500)`).
- **`ai:remove-api-key`** → `keytar.deletePassword(...)`.
- **`ai:test-connection`** → ping `claude-haiku-4-5-20251001` avec `max_tokens: 10` (~0.0001 $/test). Retourne `{ ok, error?, code? }`.
- **`ai:generate(payload)`** → appel `claude-sonnet-4-6` avec system prompt complet (~600 mots, codé en dur dans `ai.ts`). Parse le JSON renvoyé (extraction `{...}` permissive), validation `AdviceSchema` zod, retourne `{ ok, advice?, error?, code? }`.

Codes d'erreur normalisés (`AIErrorCode`) : `NO_API_KEY`, `INVALID_KEY` (HTTP 401), `RATE_LIMIT` (HTTP 429), `NETWORK`, `TIMEOUT` (30s), `BAD_RESPONSE`.

## Prompt système

Codé dans `electron/ipc/ai.ts`, environ 30 lignes. Points clés :
- Cadre : assistant pour un kinésiologue canadien.
- Insiste sur la **sélection explicite** par Marie-Eve → chercher les **liens** entre métriques, pas conseils isolés.
- Format JSON strict : `diagnostic`, `objectifsPrioritaires[]`, `programmeIntegre.{cardio,musculation,souplesse,habitudes}[]`, `echeance`, `warnings[]`.
- Règles : sobre, factuel, 3-6 items par liste, tenir compte de l'âge/sexe.
- Warnings auto : tour de taille « Très élevé » OMS → bilan médical ; âge ≥ 50 → validation cardio avant HIIT.

## Loi 25

Le payload reste anonyme par construction :
- ✅ Sexe, âge (entier), valeurs + catégories + percentiles.
- ❌ Pas de nom, courriel, date de naissance exacte, notes.

Anthropic API standard ne réutilise pas les prompts pour entraîner les modèles. Compatible Loi 25 québécoise pour un payload anonyme.

## Dette / à garder en tête

- **Pas de retry automatique** sur erreur réseau ou rate limit — voulu (évite le hammering). À l'utilisatrice de réessayer manuellement.
- **Pas de cache de réponse** : si Marie-Eve regénère avec exactement la même sélection, on rappelle l'API. Pas un problème à fréquence d'usage actuelle, mais à mémoriser si elle constate des conseils différents pour la même sélection (Claude n'est pas déterministe).
- **Timeout 30s codé en dur** : si Anthropic est lent (rare), on échoue. À ajuster si Marie-Eve voit des `TIMEOUT` injustifiés.
- **Modèle hardcodé `claude-sonnet-4-6`** : pas de UI pour switcher entre Sonnet/Opus/Haiku. Si on en a besoin (cas complexes), ajouter un dropdown dans `AIProviderCard`.
- **OpenAI déféré** : abstraction provider non faite. Si Marie-Eve veut OpenAI un jour, refactor `electron/ipc/ai.ts` pour split en `ai/anthropic.ts` + `ai/openai.ts` + dispatcher.
- **Pas de monitoring d'usage** : Marie-Eve ne voit pas combien d'appels API elle a fait dans le mois. À voir si Anthropic console suffit, sinon ajouter un compteur local.
- **`anthropic-version: 2023-06-01`** : version API hardcodée. À mettre à jour quand Anthropic en releaserà une nouvelle (rare, breaking generally well-communicated).
- **Save to notes** + **Envoyer au client par courriel** dans le modal résultat : toujours désactivés (le bouton « Envoyer au client » a un tooltip « À venir »). Prévu pour v0.1.32 quand l'onglet Notes sera branché et que le template email IA-friendly sera défini.
