# 2026-05-14 — Puissance des jambes auto (Sayers 1999) (v0.1.18)

## Travail réalisé

### Formule
Sayers SP et al. (1999), standard ACSM / SCPE pour le saut vertical :
```
Puissance_max_W = (60.7 × saut_cm) + (45.3 × poids_kg) − 2055
```
Validation avec Nicholas Jean (saut 48 cm / poids 99.8 kg) :
```
P = 60.7·48 + 45.3·99.8 − 2055 = 2913.6 + 4520.94 − 2055 = 5379.54 → arrondi 5380 W ✓
```
Match exact avec son bilan officiel.

### Implémentation
- **`src/lib/vo2max-calculator.ts`** : ajout de `sayersLegPower(verticalJumpCm, bodyWeightKg)` — retourne `null` si une donnée manque ou ≤ 0, sinon Watts arrondis.
- **`src/lib/vo2max-calculator.test.ts`** : 3 tests Sayers (cas Nicholas, cas standard 60/80, edge cases manquants).
- **`BilanData`** : nouveau flag `puissance_calculated_auto?: boolean` (env.d.ts + zod IPC + parser type).
- **`electron/lib/bilan-parser.ts`** : quand le parser .docx extrait `puissance_jambes_watts`, il marque `puissance_calculated_auto: false` pour préserver la valeur officielle.
- **`src/pages/client/BilanForm.tsx`** (`deriveBilanFields`) :
  - `flag === false` → préserver (importé).
  - `flag === undefined` + valeur présente → préserver (ancien bilan, source inconnue).
  - `flag === undefined` + valeur absente → calculer.
  - `flag === true` → recalculer à chaque save (saisie manuelle où saut/poids ont pu changer).
- **`bilanFields.ts`** : `puissance_jambes_watts` passe en `type: 'computed'` avec hint Sayers.

## Dette / à garder en tête

- **Pas de toggle UI pour overrider la valeur importée**. Le brief demandait un champ non éditable, donc pour les rares cas où Marie-Eve voudrait recalculer un ancien bilan importé : il faudrait passer par un mode debug ou ajouter un bouton « Recalculer ». À voir si le besoin remonte.
- **Hint identique pour valeurs auto et importées** : le hint sous le champ dit toujours « Sayers : 60.7 × saut + 45.3 × poids − 2055 » même pour un bilan importé où la formule peut différer (le logiciel CPAFLA utilise probablement Sayers aussi mais c'est non garanti). Améliorable : afficher « Calculé par l'app » vs « Valeur du logiciel d'origine » selon le flag.
- **Ancien bilans (Nicholas) déjà en DB sans le flag** : ils ont la valeur Sayers du logiciel CPAFLA, flag undefined, valeur présente → on préserve. Bon comportement par accident, mais si Marie-Eve modifie le saut ou le poids, la valeur ne se met PAS à jour (flag absent + valeur présente = préserver). Pour qu'elle se mette à jour, il faudrait vider d'abord `puissance_jambes_watts` — peu intuitif. **À garder en tête si elle remonte le souci.**
- **Le rapport PDF** (`ReportPage.tsx`) lit `bilan.data.puissance_jambes_watts` directement → s'affichera correctement quelle que soit la source.
