# 2026-05-14 — Saisie manuelle de bilan complet (v0.1.16)

## Travail réalisé

### Modèle de données — `BilanData` étendu (JSON, **pas de migration**)
- `met_equivalent` (MET = VO2max ÷ 3.5).
- `fc_max_predite` (Tanaka : 208 − 0.7 × âge).
- 9 champs de récupération post-effort : `recup_{1,3,5}min_{pa_sys,pa_dia,fc}`.
- `notes` (texte libre).
- Schéma `zod` côté IPC (`electron/ipc/bilans.ts`) et type côté parser (`electron/lib/bilan-parser.ts`) mis à jour.

### Architecture des calculs — split en deux modules
- **`src/lib/norms/calc.ts`** : helpers purs sans imports (BMI, MET, FC max, `categoryToScore`, `scoreToCategory`). Séparés pour rester testables avec `node --test` (qui exige des extensions `.ts` que tsc refuse hors test).
- **`src/lib/norms/scoring.ts`** : tire `calc.ts` + tables ACSM/CPAFLA, expose `computeSynthesis(data, profile)` qui retourne 6 scores composites :
  - `composition` (IMC + tour de taille)
  - `bodyFat` (% gras)
  - `aerobic` (VO2max)
  - `backHealth` (flexion + endurance dos + situps)
  - `musculoGlobal` (6 tests musculo)
  - `overall` (moyenne pondérée 25/25/25/25 composition + aérobie + dos + musculo)

## Dette / à garder en tête

- **Split `calc.ts` ↔ `scoring.ts`** : motivé par la contrainte `node --test` qui exige des extensions explicites `.ts` que tsc/vite refusent côté app. La fonction `computeSynthesis` n'est pas couverte par les tests automatisés (elle dépend des tables ACSM importées sans extension) ; elle est exercée par l'UI et indirectement par `norms.test.ts` qui valide les tables sous-jacentes.
- **Indices CPAFLA importés** (`score_composition`, `indice_sante_dos`, `score_musculo_global`, `score_global`) : encore présents dans le formulaire et la table de progression. Ils proviennent uniquement de l'import .docx — en saisie manuelle, Marie-Eve les laisse vides et la synthèse calculée prend le relais. À nettoyer si les imports finissent par disparaître.
- **% gras Durnin-Womersley auto** : remplace silencieusement la valeur saisie manuellement si les 4 plis + âge + sexe sont présents. Si Marie-Eve veut overrider, il faudra ajouter un mode « manuel » dédié. Pour l'instant : si elle ne veut pas du calcul auto, elle laisse au moins un pli vide.
- **Récupération post-effort** : pas encore catégorisée par les normes (pas de table pour la décrément FC en récupération). Champs purement informatifs pour l'instant — exploitables pour les futurs rapports.
- **Test du Score Santé Global** : pondération 25/25/25/25 (composition / aérobie / dos / musculo). Le % gras et le score `bodyFat` ne sont pas inclus dans cette moyenne pour éviter le double-comptage (le % gras est déjà composé dans « Composition corporelle » indirectement et est un test à part entière dans le brief). À confirmer avec Marie-Eve si elle veut une autre pondération.
