# 2026-05-10 (suite 2) — Retrait de fond auto sur l'avatar plein corps

## Travail réalisé

- Nouveau module `electron/lib/background-remover.ts` → `removeUniformBackground(buffer, tolerance = 30)` : lit l'image en raw RGBA via sharp, échantillonne 8 points (4 coins + 4 points à 5 px des coins) pour estimer la couleur de fond, met `alpha = 0` sur tous les pixels dans la tolérance RGB, ressort un PNG transparent. Logue la couleur détectée + le % de pixels rendus transparents.
- `electron/lib/avatars.ts` → `saveFullbodyAvatar()` applique maintenant le retrait de fond **avant** le redimensionnement final :
  1. normalise la source en Buffer + applique l'orientation EXIF (`sharp().rotate().toBuffer()`) car `removeUniformBackground` travaille sur des pixels raw ;
  2. `removeUniformBackground(oriented, 30)` ;
  3. `resize(800, 1200, { fit: inside, withoutEnlargement })` + `webp({ quality: 90, alphaQuality: 100 })` — alpha bien préservé, fichier compact.
- `saveAvatar()` (avatars circulaires) inchangée : pas besoin de détourage pour un cercle.
- Aucun changement de contrat IPC ni de schéma DB ; transparent côté renderer (`<img>` data URL WebP gère déjà l'alpha).

## À garder en tête / dette

- Hypothèse forte : **les coins de l'image sont du fond pur**. Vrai pour les illustrations DALL-E / photos studio ; faux pour une photo cadrée serrée → résultat moche (sujet mangé ou fond conservé). Idéalement : message dans l'UI à l'avenir.
- Tolérance 30 = compromis (absorbe anti-aliasing / artefacts JPEG sans manger le sujet). À ajuster si halos blancs visibles.
- Option future si contours sales : basculer sur `@imgly/background-removal-node` (modèle U-Net ~80 Mo à télécharger). Pas dans cette itération.
- ⚠️ Test manuel GUI : ré-uploader l'avatar de Nicholas Jean (homme à barbe, fond blanc) → dans Mesures il doit « flotter » sur le fond cream, sans rectangle blanc ; l'avatar circulaire (header/listes) doit être identique à avant.
