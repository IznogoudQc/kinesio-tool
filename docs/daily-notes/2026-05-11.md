# 2026-05-11 — Formulaire « Nouveau client » complet (naissance, sexe, unités)

## Travail réalisé

- Le formulaire de création (`src/pages/ClientsPage.tsx`, vue `form`) ne demande plus seulement nom + courriel : il expose désormais **tous les champs du profil** d'un coup, alignés sur le style de `EditClientModal` :
  - **Nom** \* (requis, inchangé) ;
  - **Courriel** \* (requis, inchangé) ;
  - **Date de naissance** (facultatif) — `input type="date"`, borné à `max = aujourd'hui` ;
  - **Sexe** (facultatif) — radio Femme / Homme + bouton « Effacer » quand une valeur est choisie ;
  - **Unité de longueur** — radio cm / po (pouces), défaut `cm` ;
  - **Unité de poids** — radio kg / lb (livres), défaut `kg`.
  - **Pas d'avatar à la création** — la photo (avec cropper) reste réservée à « Modifier » après coup, pour ne pas alourdir la création rapide.
- `FormState` étendu ; constante `EMPTY_FORM` (unités toujours renseignées, jamais `null`, défaut métrique) utilisée pour l'init et `openForm()`.
- **Validation client-side** : nom + courriel requis ; date de naissance facultative mais, si remplie, doit matcher `AAAA-MM-JJ` et être ≤ aujourd'hui ; sexe facultatif ; unités toujours une valeur.
- **Couche service / IPC** étendue pour accepter le payload complet :
  - `src/services/clients.ts` → `create({ name, email, birthdate?, sex?, unitLength?, unitWeight? })` ;
  - `electron/preload.ts` et `src/env.d.ts` : signature `clients.create` mise à jour ;
  - `electron/ipc/clients.ts` : `CreateClientSchema` (zod) valide `birthdate` (`IsoDateOrNull`), `sex` (`SexOrNull`), `unitLength` (`'cm' | 'in'`), `unitWeight` (`'kg' | 'lb'`) — tous optionnels ; le handler `clients:create` insère `birthdate ?? null`, `sex ?? null`, et laisse Drizzle appliquer les **défauts DB** (`cm` / `kg`) si `unitLength`/`unitWeight` sont absents.
- Pas de migration ni de changement de schéma — les colonnes existent déjà (`0006` / `0007`).
