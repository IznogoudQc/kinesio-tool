## 1. `AIAdviceContext` — état global de sélection

`src/contexts/AIAdviceContext.tsx`. Provider monté en haut du `DashboardLayout`. Expose :
- `mode: boolean` — case à cocher visible / non
- `selection: Map<string, MetricSelection>` — sélection courante
- `toggleMode()`, `toggle(key, data)`, `clear()`
- Hook `useAIAdvice()` — safe par défaut (retourne un contexte inactif si pas de provider, évite les crashs hors-dashboard)

Quand le mode est désactivé, la sélection est **automatiquement vidée** — sinon des coches invisibles continueraient d'agir.

## 2. `<MetricSelectable>` — wrapper réutilisable

`src/components/MetricSelectable.tsx`. Reçoit `selectionKey`, `data` (snapshot des infos à envoyer), `available` (cache la case si la métrique n'a pas de valeur).

Hors mode IA : rend les enfants tels quels (zero overhead visuel).
En mode IA : anneau gold + case à cocher absolute top-right. Le wrapper lui-même devient cliquable, ce qui **stoppe la propagation** vers les handlers internes (par ex. clic sur un pill timeline circ ne déclenche plus le changement de session).

## 4. `<AIAdvicePanel>` — FAB + 2 modals

**FAB (bottom-right)**, visible quand `ai.mode === true` :
- Affiche le compte de métriques sélectionnées (gold, gros).
- Bouton « Générer les conseils IA » (désactivé si 0).
- Bouton « Annuler la sélection ».
- Si 0 sélectionné : « Cochez les métriques à analyser ensemble. »

**Modal payload preview** (étape 1) :
- Liste les données qui partiront : Sexe, Âge, puis chaque métrique avec valeur + unité + catégorie + percentile.
- Bandeau gold info : « Aucune donnée identifiable n'est envoyée (pas de nom, courriel, notes). »
- [Annuler] [Confirmer et générer]

**Modal loading** (étape 2) : spinner + texte « Analyse en cours… ». Pas d'Esc pendant le loading (évite spinner orphelin).

**Modal résultat** (étape 3, wide) : 6 sections (Diagnostic, Objectifs prioritaires, Programme intégré 2×2 (cardio/muscu/souplesse/habitudes), Échéance, Avertissements). Boutons :
- **Copier en texte** ✅ fonctionnel (markdown vers presse-papier, feedback « ✓ Copié » pendant 2s)
- **Envoyer au client** — désactivé, tooltip « À venir »
- **Fermer**

Esc ferme les modals (sauf pendant loading). Body scroll bloqué quand un modal est ouvert.

## 5. `aiAdviceService` — mock

`src/services/aiAdvice.ts`. Une fonction `generate(payload): Promise<AIAdvice>` :
- Délai simulé 1.5-2.5s (réaliste vs API réelle).
- Construction d'une réponse **dynamique** basée sur les groupes de métriques détectés dans le payload :
  - Si métriques aérobie → objectifs cardio + intervalles
  - Si métriques composition → objectifs perte tour de taille + nutrition
  - Si métriques musculo → objectifs force + circuits

- Helper `formatAdviceAsText(advice)` pour le bouton Copier — produit du markdown structuré.

## Dette / à garder en tête

- **API réelle pas branchée** : le mock retourne du contenu plausible mais générique. À tester en conditions réelles avec une vraie clé Anthropic v0.1.31.
- **`MetricSelectable` stoppe la propagation des clics** quand mode IA actif. Effet voulu (cocher la card sans déclencher le timeline pill). Mais ça veut dire qu'on **ne peut pas changer de session circ** depuis la timeline si le mode IA est activé — il faut quitter le mode. Acceptable, mais à expliquer si Marie-Eve s'y prend les pieds dedans.
- **5 composites du Bilan (CompositeMiniCard) pas sélectionnables** : oversight volontaire pour limiter le scope. Si Marie-Eve veut les inclure (« génère-moi des conseils sur le score Composition globale »), 10 lignes à ajouter.
- **AIProviderCard n'est pas persistée** : le radio + l'input sont locaux à React. Au refresh, la valeur est perdue. Pas un drame avant v0.1.31, mais à valider que Marie-Eve ne s'attend pas à de la persistance.
- **`window.navigator.platform` deprecated** : utilisé pour le hint « trousseau Windows / OS » dans AIProviderCard. À remplacer par `navigator.userAgentData` ou juste retirer le hint OS spécifique.
- **Save to notes du brief** : pas implémenté, parce que l'onglet Notes est un `PlaceholderTab` (pas encore branché). À faire quand l'onglet Notes sera fait. Le bouton « Envoyer au client » sera ajouté quand l'intégration template email sera prête (v0.1.32 prévu).
- **Prompt système** : pas codé puisque c'est mocké. Quand on branchera Anthropic, suivre le squelette de l'ADR 0007 (« Tu es un assistant pour un kinésiologue canadien… programme INTÉGRÉ pas isolé »).
- **Coût API** : avec ~15 métriques × ~50 tokens = ~750 tokens prompt + ~2k tokens output. Claude Sonnet 4.5 ≈ 0.01 $ par appel. Marie-Eve fait disons 5 analyses/semaine → ~2.50 $/mois. Soutenable.
- **Bandeau d'aide IA mode actif** : affiché entre la nav et le contenu. Si Marie-Eve veut le masquer une fois l'apprentissage fait, possible d'ajouter un bouton « Compris » qui le cache via localStorage.
