## Procédures spéciales (machine sans Visual Studio C++ Build Tools)

### Premier setup (`npm install`)

Le `.npmrc` dirige `prebuild-install` vers les prebuilds Electron. Si `npm install` échoue quand même :

```bash
npm install --ignore-scripts
node node_modules/electron/install.js
npx electron-builder install-app-deps
```

### `npm run dist` — workaround winCodeSign

Sur Windows sans Developer Mode activé, `electron-builder` ne peut pas extraire le cache `winCodeSign` (symlinks macOS nécessitent `SeCreateSymbolicLinkPrivilege`).

Workaround (à exécuter une seule fois) :

```bash
python3 scripts/fix-wincodeSign-cache.py
```

Ou manuellement avec Python :

```python
import shutil, os
# Trouver un répertoire de tentative partielle dans le cache
src = r'C:\Users\njean\AppData\Local\electron-builder\Cache\winCodeSign\595473967'
dst = r'C:\Users\njean\AppData\Local\electron-builder\Cache\winCodeSign\winCodeSign-2.6.0'
if os.path.exists(dst): shutil.rmtree(dst)
shutil.copytree(src, dst)
```

**Note** : Ce workaround est permanent — une fois le répertoire `winCodeSign-2.6.0` créé dans le cache,
`npm run dist` fonctionne normalement. La solution définitive est d'activer **Windows Developer Mode**
(Paramètres → Système → Pour les développeurs → Mode développeur).

## Auto-update electron-updater — mise en place complète

### Ce qui a été fait

- **Repo public GitHub créé** : `IznogoudQc/kinesio-tool` — push initial du scaffold v0.1.0
- **`electron-updater` + `electron-log` installés** et configurés dans `electron/main.ts` :
  - Logger sur les events `checking-for-update`, `update-available`, `update-not-available`, `update-downloaded`, `error`
  - `checkForUpdatesAndNotify()` au démarrage + re-check toutes les 4h (désactivé en dev)
  - Events bridgés vers le renderer via `webContents.send('update:*')`
- **Preload étendu** (`electron/preload.ts`) : expose `window.api.app.getVersion()`, `window.api.update.quitAndInstall()` et les listeners `onChecking / onAvailable / onNotAvailable / onDownloaded / onError`
- **`electron-builder.yml`** : section `publish` ajoutée (provider github, owner IznogoudQc, repo kinesio-tool)
- **`UpdateContext.tsx`** (`src/contexts/`) : état d'update React (`idle | checking | available | not-available | downloaded | error`)
- **Affichage version en bas de sidebar** : `VersionBadge` dans `Sidebar.tsx` — couleurs adaptées selon le statut
- **Toast de mise à jour** (`UpdateToast.tsx`) : s'affiche en bas à droite quand `status === 'downloaded'`, boutons Redémarrer / Plus tard
- **GitHub Action** `.github/workflows/release.yml` : déclenche sur tag `v*`, build Windows, publie sur GitHub Releases via `GITHUB_TOKEN`
- **README.md** : section "Workflow de release" avec les 4 étapes

### Workflow de release (rappel)

```bash
npm version patch      # bump + tag local
git push --follow-tags # déclenche l'Action
```

La release apparaît sur GitHub Releases en ~5 min. Marie-Eve la détectera au prochain lancement de l'app.
