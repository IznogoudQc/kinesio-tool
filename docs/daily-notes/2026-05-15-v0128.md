# 2026-05-15 — Sélecteur de métrique en pills groupées (v0.1.28)

Refonte du sélecteur du graphique d'évolution dans `MesuresOverview`. Au lieu de 4 boutons fixes (Tour taille / hanche / Poids / % Gras), on a maintenant **21 métriques** réparties en 3 sections de pills cliquables.

## Architecture

### `MetricKey` (21 valeurs) — couvre 3 groupes :

- **Circonférences (12)** : Cou, Épaule, Biceps G/D, Poitrine, Tour taille, Abdomen, Hanche, Cuisse G/D, Mollet G/D
- **Poids & ratios (3)** : Poids, Ratio T/H, IMC
- **Composition corporelle (6)** : % Gras Siri, Somme 4 plis, Triceps, Biceps (pli), Sous-scap, Iliaque

### `MetricDef` — l'unité d'architecture
```ts
interface MetricDef {
  key: MetricKey
  label: string
  unit: string                      // 'cm' | 'in' | 'kg' | 'lb' | 'mm' | '%' | 'kg/m²' | ''
  source: 'circ' | 'plis'           // détermine la timeline + le dataset
  group: 'circ' | 'weights' | 'composition'
  accessor: (circ, plis) => number | null   // valeur brute (DB units)
  convert?: (raw) => number         // optionnel — conversion vers unité d'affichage
  lowerIsBetter: boolean
}
```

Les définitions vivent dans un `useMemo` qui dépend de `unitLength`, `unitWeight`, et `bodyHeightCm`. Tout est calculé à la volée — pas de duplication entre la liste de pills et la logique de chart.

### IMC — body height depuis les bilans
`MesureCirconferences` ne stocke pas la taille corporelle (seulement `taille` = tour de taille). Donc IMC nécessite la **hauteur** trouvée dans le bilan le plus récent qui en a une (`bilan.data.taille_cm`). On fetch les bilans au mount et on extrait la latest height. Si aucune → IMC est non-disponible (pill grisé).

### `Ratio T/H`
Calculé à la volée dans l'accessor : `taille / hanche` (en cm, indépendant de l'unité d'affichage). Précision 2 décimales.

## Tooltip enrichi

Custom `<EvolutionTooltip>` (au lieu du formatter default) :

```
┌─────────────────────────────┐
│ Cuisse G                    │
│ 64.0 cm                     │
│ 15 mai 2026                 │
│ ▼ −1.0 cm · vs 15 janv 2026 │
└─────────────────────────────┘
```

- Label de la métrique en gras
- Valeur + unité (précision : 2 décimales pour ratio T/H, 1 sinon)
- Date du point
- Delta vs précédent (avec couleur verte/rouge selon `lowerIsBetter`) + date du précédent

Le `chartData` embarque maintenant `previousValue` et `previousDate` pour chaque point (calculés en remontant les points chronologiques jusqu'au premier avec valeur non-null).

## Persistance localStorage

Clé `dashboard.mesures.selectedMetric`. Sauvegardé à chaque changement via `useEffect`. **Globale, pas par client** : Marie-Eve regarde souvent la même métrique (ex. % gras Siri) sur plusieurs clients. Lu au mount avec fallback sur `'taille'`.

## Dette / à garder en tête

- **IMC dépend de la latest bilan height** : si le client a 1 bilan avec taille 175 cm puis 1 autre avec taille 178 cm, on prendra 178 (le bilan le plus récent qui en a une). En général la taille adulte ne varie pas — acceptable.
- **`bilansService.list()` ajouté au mount** : 1 requête supplémentaire à chaque ouverture du dashboard Mesures. Faible coût (déjà fait par le sub-tab Bilan complet de toute façon — données mises en cache par le service Drizzle, mais pas par l'app). Si latence visible un jour, mémoriser.
- **`unit === ''` pour Ratio T/H** : adimensionnel. Le tooltip n'affiche pas d'unité après le nombre. À surveiller si la mise en forme est étrange (ex. « 0.91 » sans rien après).
- **`activeMetric` fallback sur `METRICS_DEFS[5]` (= Tour de taille)** : si la clé persistée n'existe plus dans une future version (rename, suppression), on retombe sur taille. Évite un crash. Mais l'URL/localStorage garde l'ancienne clé jusqu'à la prochaine sélection.
- **Section « Composition » avec 6 pills très collés** : les noms courts (« Sous-scap », « Biceps (pli) ») permettent de tenir dans la grille. Si Marie-Eve veut des noms plus explicites, gérer le débordement avec `whitespace-nowrap` + scroll horizontal au lieu de wrap.
- **Le label de la section « Plis cutanés »** dans la card existante (en bas de page) garde son nom : pas confondu avec « Composition corporelle » dans le sélecteur — c'est OK parce que ce sont deux contextes (overview vs sélecteur).
- **Custom tooltip Recharts** : remplace le default formatter pour avoir le contrôle complet du markup (date + delta + couleurs). Recharts v3 a une API stable pour `<Tooltip content={...} />` — pas de hack.
