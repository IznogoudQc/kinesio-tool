# 2026-05-15 — Mode « Synthèse » dans le sélecteur de bilan (v0.1.32)

Ajoute une pill spéciale qui affiche un bilan virtuel composé des dernières valeurs non-null pour chaque champ, agrégées à travers tous les bilans du client. Décision détaillée dans [[decisions/0009-mode-synthese-bilans]].

## Pourquoi

Les bilans importés depuis le logiciel CPAFLA actuel sont **partiels** par nature : chaque session de bilan ne remplit que les sections que Marie-Eve avait le temps de faire ce jour-là. Pour avoir « l'état le plus à jour du client », elle devait jusqu'ici parcourir bilan par bilan pour reconstituer mentalement les dernières valeurs disponibles. La synthèse fait ça automatiquement.

## Module — `src/lib/synthesisBilan.ts`

Deux fonctions exportées :

### `buildSynthesisBilan(bilans)` → `{ data, latestContributionDate, fieldOriginDates, fieldCounts }`
Parcourt les bilans dans l'ordre (le plus récent en premier, ordre naturel de `bilansService.list`). Pour chaque champ rencontré non-null/non-undefined/non-empty : si on n'a pas encore vu ce champ, on le conserve avec sa date d'origine. La date globale `latestContributionDate` est la plus récente parmi celles qui ont effectivement contribué au moins un champ.

`fieldCounts` mémorise pour chaque champ le nombre de bilans qui le renseignaient — utile si on veut désactiver le delta ▲▼ quand un champ n'a qu'une seule valeur historique (déféré pour l'instant, MeasureDelta affiche déjà « Première mesure » dans ce cas).

`fieldOriginDates` est retourné mais pas encore utilisé côté UI — prévu pour les tooltips d'origine en v0.1.33.

### `buildPreviousSynthesisBilan(bilans)` → `{ data }`
Pour chaque champ, conserve la **2e** valeur non-null rencontrée. Sert aux flèches ▲▼ champ par champ : un même affichage de delta peut mélanger des comparaisons venant de bilans différents (VO2max actuel sept 2025 vs précédent oct 2024 = delta sur 11 mois ; IMC actuel janv 2025 vs précédent juin 2023 = delta sur 19 mois). C'est la nature même de la synthèse.

### Tests
7 cas couverts :
- 1 seul bilan → copie identique
- 2 bilans avec recouvrement partiel → préférence du plus récent par champ
- null / undefined / `''` ignorés
- `0` numérique valide (pas filtré)
- `fieldCounts` correct
- `buildPreviousSynthesisBilan` prend bien la 2e valeur
- Liste vide → résultat vide

## UI — `BilanSelectorPills`

- Pill spécial en tête, icône `Sparkles` lucide.
- Style actif : `bg-gradient-to-br from-marine to-marine-light text-cream border-gold shadow-md ring-2 ring-gold/40`.
- Style inactif : `bg-gold/5 text-marine border-gold/30` (différencié des pills bilans normaux).
- Sous-titre : « MAJ + date » si `synthesisLatestDate` fournie, sinon « Dernières valeurs ».
- Type modifié : `selectedId: string | null` ; `onSelect` reçoit `null` pour synthèse.

## Routing

- Sentinel `?bilan=synthesis` dans l'URL.
- **Nouveau défaut** : si pas de paramètre, mode synthèse.
- Click sur la pill Synthèse → `?bilan=synthesis` dans l'URL (bookmarkable).
- Click sur le bouton « Revenir à la synthèse » (anciennement « Revenir au bilan récent ») → retire le param URL → mode synthèse via défaut.

## DashboardTab — adaptations

```ts
const isSynthesisMode = selectedBilanIdFromUrl === null || selectedBilanIdFromUrl === 'synthesis'
const synthesisResult = useMemo(() => buildSynthesisBilan(bilans ?? []), [bilans])
const previousSynthesisResult = useMemo(() => buildPreviousSynthesisBilan(bilans ?? []), [bilans])

const activeBilan = useMemo(() => {
  if (isSynthesisMode && synthesisResult) {
    return {
      id: 'synthesis',
      clientId: client.id,
      date: synthesisResult.latestContributionDate ?? bilans[0].date,
      data: synthesisResult.data,
      source: 'manuel',
      createdAt: new Date().toISOString()
    }
  }
  // …logique existante…
}, [...])
```

Les composants downstream (`StatCardXL`, `MusculoRadar`, `CompositeMiniCard`, etc.) traitent ce bilan virtuel comme un bilan réel — pas de changement à faire chez eux. Le `BilanData` du bilan virtuel contient bien les champs attendus.

`isViewingOlder` (bandeau « vous consultez un bilan ancien ») reste `false` en mode synthèse.

Header : remplace « Bilan affiché : DATE » par « 🔬 Synthèse — dernières valeurs disponibles · mise à jour DATE · N bilans agrégés ».

## Dette / à garder en tête

- **Tooltips d'origine pas encore branchés** : `fieldOriginDates` est calculé mais le UI ne le consomme pas. Marie-Eve ne voit pas explicitement quelle date a contribué chaque valeur. À ajouter en v0.1.33 si l'opacité devient une frustration : tooltip au hover de chaque card « extrait du bilan du 4 sept 2025 ».
- **Comparaisons cross-bilans** : les deltas ▲▼ mélangent des comparaisons venant de bilans différents — VO2max sur 11 mois, IMC sur 19 mois. C'est intentionnel mais l'utilisatrice ne le sait pas explicitement. À documenter avec un texte d'aide au-dessus du dashboard en mode synthèse (« Les comparaisons ▲▼ portent sur la valeur précédente connue, qui peut dater de différents bilans »).
- **`activeBilan.id === 'synthesis'`** : passé à `ProgressionChart` qui s'attend à un ID matchant un bilan réel — aucun point n'est highlighted sur la frise en mode synthèse. C'est OK sémantiquement (la synthèse n'a pas un point unique sur l'axe temps) mais perte de feedback visuel. Si Marie-Eve veut un highlight, on pourrait marquer le **dernier** point de la frise (= bilan le plus récent contributeur) — pas urgent.
- **Pas de cache du résultat de `buildSynthesisBilan`** : recalculé à chaque render du DashboardTab. `useMemo` sur `bilans` limite le recalcul aux changements de la liste — OK pour N bilans typiques < 50, mais quadratique côté nombre de champs × nombre de bilans. Si on a un client avec 200 bilans (improbable), à optimiser.
- **Synthèse de la synthèse** : si Marie-Eve voulait un jour comparer 2 « périodes synthétiques » (par ex. synthèse 2024 vs synthèse 2025), il faudrait ajouter un sélecteur de période côté synthèse. Pas demandé pour v0.1.32.
- **Pas de UI pour voir le détail des contributions** : Marie-Eve ne voit pas explicitement « 7 champs viennent du bilan A, 3 du bilan B, 5 du bilan C ». Si elle veut comprendre la composition, elle doit cliquer sur les bilans un par un. Tooltips d'origine (v0.1.33) couvriraient ça.
