# 2026-05-15 — Corrections du rapport PDF (v0.1.37)

5 correctifs sur le rapport PDF refondu en v0.1.36, après un premier rendu réel.

## 1. Couverture — avatar carré (visage)

`CoverPage` utilisait `avatarFullbodyFilename` (plein corps) → on voyait le torse dans le cercle. Bascule sur `avatarFilename` (carré, recadré sur le visage). Fallback : avatar carré → silhouette sex-aware (`body-male.png` / `body-female.png` en `object-fit: contain`) → cercle crème vide.

## 2. Bug page-break — Section 5 « En détail »

Les blocs métriques étaient coupés en plein milieu entre pages. Chaque bloc reçoit `className="break-inside-avoid"` (Tailwind → `break-inside: avoid`, supporté par le moteur de print Chromium de `printToPDF`). `perPage` passe de 5 à 4 pour réduire le débordement à la source.

## 3. Barèmes cliniques — PA + FC repos

PA systolique/diastolique et FC repos affichaient « Pas de barème normatif ». Nouveau fichier `src/lib/norms/clinical.ts` :

- **PA** (seuils OMS/JNC, universels) : Optimale / Normale / Pré-HT / HT1 / HT2 → mappés sur les 5 catégories ACSM (EXCELLENT → A_AMELIORER).
- **FC repos** (ACSM) : base hommes 36-45 ans, écart femmes ≈ +3 bpm. **Indépendant de l'âge** — les normes FC repos varient de ≈ 2-3 bpm sur l'âge adulte, simplification assumée.

Modélisés en `NormRange` (`lowerIsBetter: true`, percentiles décroissants) pour réutiliser tel quel le moteur de catégorisation. Branchés via `getRange` dans `norms/index.ts` : les seuils cliniques priment et court-circuitent les tables ACSM/CPAFLA. 3 `TestKey` ajoutés (`bloodPressureSystolic`, `bloodPressureDiastolic`, `restingHeartRate`), entrées `null` correspondantes dans `acsm.ts` (le `Record<TestKey, …>` est exhaustif), mappings ajoutés dans `bilan-keys.ts`. `getCategorization` / `getNormPercentiles` / `getPercentile` / `getNextCategoryTarget` fonctionnent automatiquement (tous passent par `getRange`).

## 4. Phrases explicatives par catégorie

Les 2 phrases boilerplate répétées partout sont remplacées par `EXPLANATION_BY_CATEGORY` : une phrase par catégorie (5 distinctes). Aucune phrase si la métrique n'est pas catégorisée.

## 5. Axes des graphes

- XAxis des 8 graphes (pages 4-5) : `interval="preserveStartEnd"` + `tickMargin={6}` — labels lisibles, plus collés.
- Graphes de score (musculo, santé du dos) : axe Y forcé `domain={[0,5]}` + `ticks={[0,1,2,3,4,5]}`. `SingleLineChart` reçoit une prop `scoreAxis` pour ça (réutilisé hors-score avec axe auto).
